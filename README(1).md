# MVP. Автоматическая классификация типов математических ошибок

Магистерская диссертация Мелконяна О. А.
Программа «Электронный бизнес и цифровые инновации», ВШЭ

## Состав

- `MVP_Colab_Notebook.ipynb` – основной Google Colab-ноутбук с обучением классификатора, визуализацией и демонстрацией пайплайна на тестовых сценариях.
- `app.py` – Streamlit-приложение для интерактивной демонстрации пайплайна.
- `requirements.txt` – зафиксированные версии библиотек.

## Запуск в Google Colab (рекомендуемый способ)

1. Открыть `MVP_Colab_Notebook.ipynb` в Google Colab.
2. Нажать `Runtime → Run all`.
3. Полный прогон 12–15 минут на бесплатном tier без GPU.

Ноутбук на выходе сохраняет в `/content/`:
- `best_classifier.joblib` – обученный классификатор
- `recommendations.json` – словарь педагогических рекомендаций
- `confusion_matrix.png` – визуализация confusion matrix
- `umap_2d.png` – UMAP-проекция эмбеддингов
- `embeddings_e5_demo.npy` – эмбеддинги демо-датасета

## Запуск Streamlit-демо локально

```bash
pip install -r requirements.txt
# Сначала запустить Colab-ноутбук и скачать best_classifier.joblib и recommendations.json
streamlit run app.py
```

Браузер откроется на `http://localhost:8501`.

## Архитектура MVP

```
ученическое решение
     ↓
preprocessing (нормализация нотации)
     ↓
multilingual-e5-large векторизация (1024-мерный embedding)
     ↓
ансамбль классификаторов sklearn (SVM + RF + LogReg)
     ↓
голосование по большинству + расчёт confidence
     ↓
если confidence ≥ 0.8 → прямой ответ (label + рекомендация)
если confidence < 0.8 → эскалация к LLM-fallback (GPT-4o/GigaChat/DeepSeek)
     ↓
выход. ярлык типа ошибки + текстовая рекомендация
```

## Демо-датасет

Внутри ноутбука содержится 20 размеченных примеров (по 5 на каждый из 4 типов ошибок). Этого достаточно для демонстрации работы пайплайна и проверки воспроизводимости.

Полный датасет на 247 примеров с межэкспертным согласием κ = 0,74 доступен в репозитории работы при предъявлении подтверждения принадлежности к академическому сообществу (для предотвращения утечки экспертно-размеченного контента).

## Поддерживаемые типы ошибок

1. **Концептуальные.** Математический объект обработан неверно по существу.
2. **Вычислительные.** Метод верный, арифметический результат неверный.
3. **Процедурные.** Порядок шагов алгоритма нарушен.
4. **Невнимательность.** Описки, подмены, пропуски.
