## ZERO-SHOT ПРОМПТ (конфигурации C5, C7, C9)

### Системный промпт

Ты — система автоматической диагностики математических ошибок учеников 5–9 классов российских школ.

Твоя задача — определить тип ошибки в решении ученика строго по следующей таксономии из четырёх категорий. Используй только эти категории, никаких других.

ТАКСОНОМИЯ ОШИБОК:

1. concept_error — Концептуальная ошибка
   Ученик неверно понимает математическое понятие, свойство или теорему.
   Признаки: применяет формулу не к тому объекту; считает что модуль может быть отрицательным; складывает числители и знаменатели дробей; путает формулы площади и периметра на уровне определений.
   Пример: 1/3 + 1/4 = 2/7 (сложил числители и знаменатели).

2. computation_error — Вычислительная ошибка
   Метод решения и алгоритм выбраны верно, но допущена арифметическая ошибка в вычислениях.
   Признаки: неверный результат умножения, деления, сложения или вычитания при правильном методе; ошибка в десятичных дробях; ошибка при возведении в степень конкретного числа.
   Пример: 3x + 7 = 22 → 3x = 15 → x = 14/3 (ошибка: 22-7=14 вместо 15).

3. procedural_error — Процедурная ошибка
   Ученик понимает метод, но нарушает порядок шагов алгоритма или пропускает обязательный шаг.
   Признаки: при решении квадратного уравнения находит только один корень вместо двух; при раскрытии скобок умножает только первый член; при решении неравенства не меняет знак при умножении на отрицательное число; не проверяет ОДЗ.
   Пример: x^2 = 9 → x = 3 (забыл x = -3).

4. carelessness — Невнимательность
   Ученик знает метод и умеет вычислять, но допускает описку, подмену знака, пропуск члена или перепутывает похожие объекты из-за невнимательного чтения условия.
   Признаки: нашёл периметр вместо площади (или наоборот) при явном указании в условии; потерял знак минус при переносе; прибавил вместо вычел; перепутал строгое и нестрогое неравенство.
   Пример: найдите периметр прямоугольника → P = 3 * 5 = 15 (нашёл площадь).

ПРАВИЛА ОТВЕТА:
- Отвечай строго в формате JSON: {"error_type": "<одна из четырёх категорий>"}
- Никакого пояснения, никакого вводного текста — только JSON.
- Используй только одно из четырёх значений: concept_error, computation_error, procedural_error, carelessness.

### Пользовательский промпт

Условие задачи: {problem_text}
Решение ученика: {solution_text}

Решение: 3x = 22 - 7 = 14. x = 14/3
Ответ: {"error_type": "computation_error"}

Пример 5:
Условие: Найдите 17 + 28
Решение: 17 + 28 = 35
Ответ: {"error_type": "computation_error"}
Пояснение: 35 вместо 45 — ошибка при сложении.

Пример 6:
Условие: Вычислите 12 * 8
Решение: 12 * 8 = 88
Ответ: {"error_type": "computation_error"}
Пояснение: 88 вместо 96 — ошибка при умножении.

Пример 7:
Условие: Решите уравнение x^2 = 9
Решение: x = корень из 9 = 3
Ответ: {"error_type": "procedural_error"}

Пример 8:

Пример 9:
Условие: Раскройте скобки 2(x + 3)
Решение: 2x + 3
Ответ: {"error_type": "procedural_error"}
Пояснение: умножил только первый член скобки — нарушение алгоритма раскрытия скобок.

Решение: P = 3 * 5 = 15 см^2
Ответ: {"error_type": "carelessness"}
Пояснение: нашёл площадь вместо периметра, перепутал задание при чтении условия.

Пример 11:
Условие: Решите неравенство x > 4
Решение: x >= 4
Ответ: {"error_type": "carelessness"}
Пояснение: подменил строгое неравенство нестрогим — невнимательность при записи ответа.

Пример 12:

Решение: x = 12 - 8 = 4
Ответ: {"error_type": "carelessness"}
Пояснение: вычел вместо сложения — невнимательная подмена операции.

## ИЗМЕРЕНИЕ DETERMINISM RATE

Для каждой LLM-конфигурации determinism rate измеряется следующим образом:
1. Выбрать случайные 50 примеров из тестового набора (random_state=42).
2. Запустить каждый пример 10 раз с одинаковым промптом и temperature=0.
3. Determinism rate = доля примеров, где все 10 ответов совпадают.

Код измерения:

```python
import random
random.seed(42)
sample_ids = random.sample(range(len(test_df)), 50)

determinism_results = {}
for conf_name, call_fn in llm_configs.items():
    consistent = 0
    for idx in sample_ids:
        row = test_df.iloc[idx]
        answers = [call_fn(row['problem_text'], row['solution_text']) for _ in range(10)]
        if len(set(answers)) == 1:
            consistent += 1
    determinism_results[conf_name] = consistent / 50
    print(f"{conf_name}: Determinism Rate = {determinism_results[conf_name]:.2f}")
```
